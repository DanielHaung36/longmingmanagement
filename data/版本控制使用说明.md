# Excel版本控制使用说明

## 🎯 核心原理

每次对项目进行**新增、修改、删除**操作时，系统会自动：
1. 更新Excel文件
2. 记录变更日志到 `excel-changelog.json`
3. 生成版本号（v1, v2, v3...）

## 📝 自动记录的信息

每条日志包含：
- **id**: 唯一编号
- **action**: 操作类型（CREATE/UPDATE/DELETE）
- **projectCode**: 项目编号
- **timestamp**: 操作时间（ISO格式）
- **version**: 版本号
- **details**: 详细信息（包含修改前后的数据）

## 📊 实际例子

### 场景1: 创建新项目

```javascript
// 代码里你只需要调用创建项目的API
POST /api/projects/create
{
  "name": "宝钢铁矿试验项目",
  "jobType": "AT",
  "clientCompany": "宝钢铁矿",
  ...
}

// 系统会自动记录到 excel-changelog.json:
{
  "id": 1,
  "action": "CREATE",
  "projectCode": "2024-AT-BXT-001",
  "timestamp": "2024-10-03T02:15:30.123Z",
  "version": "v1",
  "details": {
    "rowNumber": 5,           // Excel第5行
    "projectId": 12,          // 数据库ID
    "data": {
      "jobType": "AT",
      "clientCompany": "宝钢铁矿",
      "projectCode": "2024-AT-BXT-001"
    }
  }
}
```

### 场景2: 修改项目信息

```javascript
// 更新项目API
PATCH /api/projects/12
{
  "clientCompany": "宝钢铁矿有限公司"  // 修改公司名
}

// 自动记录变更（注意记录了修改前后的数据）:
{
  "id": 3,
  "action": "UPDATE",
  "projectCode": "2024-AT-BXT-001",
  "timestamp": "2024-10-03T05:45:22.789Z",
  "version": "v3",
  "details": {
    "rowNumber": 5,
    "projectId": 12,
    "oldData": {              // ⭐ 修改前
      "clientCompany": "宝钢铁矿"
    },
    "newData": {              // ⭐ 修改后
      "clientCompany": "宝钢铁矿有限公司"
    }
  }
}
```

### 场景3: 删除项目

```javascript
// 删除项目API
DELETE /api/projects/12

// 自动记录删除操作:
{
  "id": 5,
  "action": "DELETE",
  "projectCode": "2024-AT-BXT-001",
  "timestamp": "2024-10-03T10:30:00.567Z",
  "version": "v5",
  "details": {
    "rowNumber": 5,
    "projectId": 12
  }
}
```

## 🔍 如何查看版本历史

### 方法1: 通过API查看

```bash
# 1. 查看某个项目的所有变更记录
curl http://localhost:8080/api/excel/changelog/2024-AT-BXT-001

# 返回结果（按时间倒序）:
{
  "success": true,
  "projectCode": "2024-AT-BXT-001",
  "data": [
    {
      "id": 5,
      "action": "DELETE",
      "version": "v5",
      "timestamp": "2024-10-03T10:30:00.567Z",
      ...
    },
    {
      "id": 3,
      "action": "UPDATE",
      "version": "v3",
      "timestamp": "2024-10-03T05:45:22.789Z",
      ...
    },
    {
      "id": 1,
      "action": "CREATE",
      "version": "v1",
      "timestamp": "2024-10-03T02:15:30.123Z",
      ...
    }
  ],
  "total": 3
}

# 2. 查看最近50条变更
curl http://localhost:8080/api/excel/changelog?limit=50

# 3. 查看当前Excel版本号
curl http://localhost:8080/api/excel/version
# 返回: {"success": true, "version": "v5"}
```

### 方法2: 直接查看日志文件

打开文件：`data/excel-changelog.json`

```json

    "timestamp": "2024-10-03T02:15:30.123Z",
    "version": "v1",
    "details": { ... }
  },
  ...
]
```

## 🎨 在前端展示版本历史

可以做一个时间线组件：

```typescript
// React组件示例
function ProjectVersionHistory({ projectCode }) {
  const [history, setHistory] = useState([]);

  useEffect(() => {
    fetch(`/api/excel/changelog/${projectCode}`)
      .then(res => res.json())
      .then(data => setHistory(data.data));
  }, [projectCode]);

  return (
    <Timeline>
      {history.map(log => (
        <Timeline.Item key={log.id} color={
          log.action === 'CREATE' ? 'green' :
          log.action === 'UPDATE' ? 'blue' : 'red'
        }>
          <p>{log.version} - {log.action}</p>
          <p>{new Date(log.timestamp).toLocaleString()}</p>
          {log.action === 'UPDATE' && (
            <div>
              <p>修改前: {JSON.stringify(log.details.oldData)}</p>
              <p>修改后: {JSON.stringify(log.details.newData)}</p>
            </div>
          )}
        </Timeline.Item>
      ))}
    </Timeline>
  );
}
```

## 📈 实际应用场景

### 1. 追溯项目变更
"老板，这个项目的公司名什么时候改的？"
→ 查看changelog，立即找到v3版本的修改记录

### 2. 数据审计
"上个月谁删除了这个项目？"
→ 查看DELETE记录，包含操作时间和详情

### 3. 错误恢复
"我不小心改错了，原来的数据是什么？"
→ 查看UPDATE记录的oldData字段

### 4. 统计分析
"最近7天有多少项目变更？"
→ 过滤changelog的timestamp字段

## 🔧 技术细节

### 日志文件位置
```
data/excel-changelog.json
```

### 何时记录日志
每次调用以下函数时自动记录：
- `ExcelService.syncProjectCreate()`
- `ExcelService.syncProjectUpdate()`
- `ExcelService.syncProjectDelete()`

### 日志文件结构
```json
[
  { "id": 1, "version": "v1", ... },
  { "id": 2, "version": "v2", ... },
  { "id": 3, "version": "v3", ... }
]
```

版本号自动递增，永不重复。

## 💡 常见问题

**Q: 日志文件会不会太大？**
A: 可以定期归档，比如每月压缩一次历史日志。

**Q: 能不能回滚到某个版本？**
A: 当前只记录历史，不支持自动回滚。可以手动参考oldData恢复。

**Q: 删除的项目还能找回来吗？**
A: 是软删除，Excel行还在（标记为灰色），changelog里也有完整记录。

**Q: 如何清理旧日志？**
A: 可以定期将excel-changelog.json移动到backups/目录，重新开始计数。
